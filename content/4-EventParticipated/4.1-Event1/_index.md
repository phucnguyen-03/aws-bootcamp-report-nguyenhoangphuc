# Event Report – AWS Cloud Knowledge Sharing Session

### Purpose of the Event

- Share practical experience in using AWS Cloud services.
- Introduce AWS Best Practices for designing and deploying cloud solutions.
- Improve learning strategies, teamwork, and professional development skills.
- Share effective time management techniques and ways to overcome procrastination in daily work.

### Organizers

- AWS Internship Program
- AWS Mentors and Cloud Engineers

### Key Topics

#### Practical Experience with AWS Cloud

- Introduction to the workflow of developing and deploying applications on AWS.
- Sharing practical experience in selecting AWS services for different business requirements.
- Best practices for improving system performance while optimizing cloud costs.
- Introduction to Cloud-Native architecture and scalable system design.

#### AWS Best Practices

- Designing serverless architectures where appropriate.
- Applying the Principle of Least Privilege when configuring IAM permissions.
- Monitoring cloud costs using AWS Budgets and Amazon CloudWatch.
- Organizing AWS resources by separating Development and Production environments.

#### Project Development Experience

- Planning and organizing project tasks effectively.
- Tracking project progress through milestones.
- Communicating proactively with mentors whenever issues arise.
- Understanding the importance of continuous testing and regular progress reviews.

#### Professional Development Skills

- Effective time management techniques.
- Developing a proactive mindset for learning and working.
- Overcoming procrastination by breaking large tasks into smaller achievable goals.
- Continuously learning and keeping up with the latest cloud technologies.

### Knowledge Gained

#### Technical Knowledge

- Gained a better understanding of how organizations build and operate systems on AWS Cloud.
- Learned how to select appropriate AWS services based on different project requirements.
- Understood important cloud architecture principles focusing on scalability, security, performance, and cost optimization.
- Recognized the importance of designing maintainable and highly available cloud systems.

#### Working Skills

- Improved project planning and task management skills.
- Enhanced communication and collaboration with mentors and teammates.
- Learned to become more independent in researching documentation and solving technical problems.

#### Personal Development

- Realized the negative impact of procrastination on project schedules.
- Learned practical methods for planning study and work activities more efficiently.
- Understood that continuous learning is essential for staying current in the rapidly evolving cloud computing industry.

### Applying the Knowledge to My Project

After attending this knowledge-sharing session, I applied several ideas to my **AI-Powered Smart Document Assistant** project, including:

- Designing the system using a Serverless Architecture on AWS.
- Applying AWS Best Practices when selecting cloud services.
- Building a development roadmap with clearly defined milestones.
- Monitoring cloud spending using AWS Budgets.
- Regularly discussing project progress with mentors and incorporating their feedback.
- Breaking down project tasks into weekly objectives to improve productivity and avoid procrastination.

### Event Experience

The AWS Cloud Knowledge Sharing Session provided valuable insights from mentors and cloud engineers with real-world industry experience. The event focused not only on technical topics but also on professional working methods, learning strategies, and career development within the cloud computing field.

Through practical examples and real project experiences, I gained a better understanding of how to select appropriate AWS services, design cloud-native architectures, and optimize both system performance and operational costs. In addition, the discussions on time management, proactive learning, and overcoming procrastination helped me improve the way I organize and execute my internship project.

Another valuable takeaway from the event was realizing that technical knowledge alone is not enough to become a successful software engineer. Communication, teamwork, continuous learning, and the ability to adapt to new technologies are equally important for long-term career growth.
